Rules for Golfing

Spaghetti is NISER Coding Club's platform for conducting (recreational) programming contests. Currently, Spaghetti only supports Code Golf.

Rules for Code Golf

  1. Solutions that have the lowest character count win (duh). In case of solutions with equal character counts, the one submitted earlier is preferred.
  2. For challenges, that require an input, it can either be taken from the standard input (stdin/keyboard), or can be passed as argument(s) to a function. Outputs can be return value to a function, or be printed to the console (standard output/stdout). Note that, your program must be invokable from the command line, either the system console or a language-specific CLI (e.g. in case of MatLab or Mathematica Scripts, where even user-defined functions /classes can be called directly from the CLI) and it must return the output to the console/CLI.
  3. Almost all standard programming languages are allowed. Esolangs and languages made specifically for golfing are forbidden.
  4. Loopholes that are 'forbidden by default' on Code Golf Stack Exchange, are forbidden on Spaghetti. These include, but are not limited to: interpreting the challenge too literally, fetching result/code from an external source, and using made up languages designed specifically for the challenge.
