Rules for Golfing

Spaghetti is NISER Coding Club's platform for conducting (recreational) programming contests. Currently, Spaghetti only supports Code Golf.

Rules for Code Golf

  1. Solutions that have the lowest character count win (duh). In case of solutions with equal character counts, the one submitted earlier is preferred.
  2. For challenges, that require an input, it can either be taken from the standard input (stdin/keyboard), or can be passed as argument(s) to a function. Outputs can be return value to a function, or be printed to the console (standard output). Note that, you must provide compilation and execution instructions with your submission for it to be considered valid (see next point).
  3. Compilation/execution instructions must be specified for every submission. At the very least mention the following:
    1. Compiler/interpreter version you used
    2. Relevant function(s) that need(s) to be called
    3. Method by which the program accepts input
  4. Almost all standard programming languages are allowed. Esolangs and languages made specifically for golfing are forbidden.
  5. Loopholes that are 'forbidden by default' on Code Golf Stack Exchange, are forbidden on Spaghetti. These include, but are not limited to: interpreting the challenge too literally, fetching result/code from an external source, and using made up languages designed specifically for the challenge.
