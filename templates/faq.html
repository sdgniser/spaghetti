Frequently Asked Questions

What is Spaghetti? Why Spaghetti?

Spaghetti is the name we have given to our tournament platform.

I'll let Wikipedia explain the why:

Spaghetti code is a pejorative phrase for unstructured and difficult-to-maintain source code. Spaghetti code can be caused by several factors, such as volatile project requirements, lack of programming style rules, and insufficient ability or experience.

Why is this site 'insecure'?

Browsers mark this site as insecure because we use a self signed SSL certificate, that has not been signed by a certificate authority (CA). Certificate Authorities require a registered public domain to issue a signature on the certificate, and since we are not hosting this platform on the internet (yet), it is not possible to get our certificate verified.

The self signed certificate, however, is much more secure than plain HTTP. The communication between your web browser and our server happens over an encrypted channel, and the data is at risk only if someone gets access to our server (or mounts a Man in the Middle attack).

What programming languages/libraries is Spaghetti built on?

The backend has been written with Django (a Python web framework), and PostgreSQL. The frontend currently uses awsm.min for styling.

I spotted a bug! What do I do?

Inform us about the bug!

Write to us at code@niser.ac.in or sdg@niser.ac.in with details of the bug, or open an issue on GitHub.

It would be great if X was added to the platform...

Write to us with details of X at code@niser.ac.in or sdg@niser.ac.in or open an issue on GitHub.

I want to contribute to the project! Where do I start?

There are multiple ways to contribute! If you do not have experience with coding you can help us with testing new features before they are pushed to production. If you want to contribute with code, you can contact us at code@niser.ac.in or sdg@niser.ac.in. We don't have a style guide in place yet, so we would discourage making PRs without informing us about them first.
